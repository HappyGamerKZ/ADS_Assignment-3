# Assignment 3: HashTable and Binary Search Tree (BST)

Данный проект содержит реализацию двух структур данных: **Хэш-таблицы** (метод цепочек) и **Бинарного дерева поиска** (BST). Проект выполнен в рамках учебного задания по алгоритмам и структурам данных (ADS).

## ⚠️ Основное ограничение
В соответствии с требованиями задания, все алгоритмы (вставка, поиск, удаление, обход) реализованы **без использования рекурсии** (исключительно циклы `while` и вспомогательный стек).

---

## 🛠 Реализация

### 1. MyHashTable (Хэш-таблица)
Реализована на основе массива связных списков (**Separate Chaining**).
* **Класс ключа:** Используется `MyTestingClass`.
* **Custom HashCode:** Реализован собственный алгоритм хэширования без использования стандартных методов Java (таких как `Objects.hash()` или `String.hashCode()`).
* **Равномерное распределение:** Алгоритм настроен таким образом, чтобы при добавлении 10,000 элементов распределение по бакетам было максимально равномерным.

### 2. MyBST (Бинарное дерево поиска)
Реализовано с поддержкой итератора и отслеживанием размера.
* **Size:** Переменная обновляется при каждой операции вставки/удаления (сложность $O(1)$).
* **Итератор:** Реализован `In-order traversal` (симметричный обход) с использованием `java.util.Stack`. Элементы выводятся в порядке возрастания ключей.
* **Доступ к данным:** Во время итерации обеспечен доступ как к ключу (`getKey()`), так и к значению (`getValue()`) узла.

## 📈 Результаты тестирования

### Распределение HashTable (10,000 элементов, M=11):
При запуске тестов с 10,000 уникальными ключами хэш-функция показала следующие результаты (пример):
- **Бакет 0:** 910
- **Бакет 1:** 908
- ...
- **Бакет 10:** 911

Разброс между бакетами минимален, что подтверждает выполнение условия **Uniform Distribution**.

### Тест BST:
Дерево корректно выполняет итерацию. Для набора ключей `{5, 3, 8, 2, 4}` итератор выводит:
`2 -> 3 -> 4 -> 5 -> 8`.
